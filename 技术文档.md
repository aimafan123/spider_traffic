## 1. 项目概述

### 1.1 项目背景与目标

Spider Traffic 是一个集成了网络爬虫与流量捕获功能的高度自动化框架。该项目旨在为网络行为分析、安全研究和性能评测提供强大的数据支持。

**项目目标：**
- 提供全自动化的网络流量捕获和分析解决方案
- 支持多种代理模式下的流量采集（direct/xray/tor）
- 实现 TLS 加密流量的自动解密和解析
- 为网络安全研究和性能分析提供标准化的数据格式
- 支持大规模网站批量分析和数据采集

### 1.2 核心功能说明

#### 1.2.1 网络爬取与流量采集
- **全自动工作流**：从启动浏览器到关闭进程，全程无需人工干预
- **多代理模式支持**：
  - `direct`：直连模式，适用于常规流量分析
  - `xray`：代理模式，支持复杂路由和协议转换
  - `tor`：匿名网络模式，基于Tor网络进行访问
- **精准的流量捕获**：使用 tcpdump 捕获指定网络接口的全部流量
- **智能链接提取**：自动提取页面链接并进行深度爬取
- **反爬虫机制**：集成随机延时、用户代理轮换等反检测机制

#### 1.2.2 TLS 流量解密与解析
- **自动化 HTTP/2 解码**：利用 SSLKEYLOGFILE 自动解密 TLS 流量
- **HTTP/1.1 支持**：同时支持传统 HTTP/1.1 协议解析
- **结构化数据输出**：提取详细请求数据并输出为 JSON 格式
- **多协议兼容**：支持不同版本的 HTTP 协议和 TLS 版本
- **Trojan协议解码**：专门支持 Trojan 代理协议的流量解密

#### 1.2.3 任务管理与调度
- **URL任务队列**：支持批量URL处理和任务调度
- **进程生命周期管理**：自动管理浏览器和代理进程
- **状态持久化**：支持任务状态保存和恢复
- **错误恢复机制**：自动处理网络异常和进程崩溃

### 1.3 技术栈介绍

**核心技术栈：**
- Python 3.12 + Scrapy 2.12.0（网络爬虫框架）
- Playwright 1.49.1（浏览器自动化）
- PyShark 0.6（网络数据包分析）
- Docker（容器化部署）
- Xray-core（代理服务器）
- Tor（匿名网络）

**依赖库详解：**
- `scrapy-playwright`：Scrapy与Playwright集成
- `pyshark`：基于Wireshark的Python包分析库
- `scapy`：网络数据包操作库
- `paramiko`：SSH连接和文件传输
- `beautifulsoup4`：HTML解析库

## 2. 系统架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Spider Traffic 系统                      │
├─────────────────────────────────────────────────────────────┤
│  任务管理(Task) │ 配置管理(Config) │ 日志系统(Logger)        │
├─────────────────────────────────────────────────────────────┤
│                 核心控制模块 (Main/Action)                   │
│  爬虫启动 │ 流量捕获 │ 进程管理 │ 解码调度                  │
├─────────────────────────────────────────────────────────────┤
│  Scrapy爬虫引擎 │ Playwright中间件 │ TLS解密模块             │
├─────────────────────────────────────────────────────────────┤
│  网络层: Direct模式 │ Xray模式 │ Tor模式                    │
├─────────────────────────────────────────────────────────────┤
│  数据输出: PCAP文件(原始流量) │ JSON解析结果 │ 日志文件     │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 数据流架构

```
URL任务队列 → 浏览器启动 → 流量捕获开始 → 页面访问 → 链接提取
     ↓              ↓              ↓           ↓          ↓
任务调度器 → 进程管理器 → tcpdump进程 → Scrapy引擎 → 深度爬取
     ↓              ↓              ↓           ↓          ↓
状态保存 → 资源清理 → PCAP文件生成 → 流量解密 → JSON输出
```

## 4. 部署指南

### 4.1 环境要求

**系统要求：**
- Linux (Ubuntu 20.04+, Debian 12+)
- 内存: 最低4GB，推荐8GB+
- 存储: 最低20GB可用空间
- Python 3.12+, Docker 20.10+

**网络要求：**
- 具备root权限（用于tcpdump流量捕获）
- 网络接口访问权限
- 防火墙配置允许目标端口访问

### 4.2 安装步骤


```python
def validate_url(url: str) -> bool:
    """验证URL格式和安全性"""
    if not url or not isinstance(url, str):
        return False
    
    # 检查协议
    if not url.startswith(('http://', 'https://')):
        return False
    
    # 检查恶意字符

    if any(char in url for char in dangerous_chars):
        return False
    
    return True
```
